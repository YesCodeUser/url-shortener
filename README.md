# 🔗 URL Shortener Microservice

Высокопроизводительный асинхронный микросервис для сокращения ссылок, построенный на стеке **FastAPI** + **SQLAlchemy 2.0** + **PostgreSQL**.

---

## ✨ Ключевые особенности и решения

*   **Асинхронность:** Полностью асинхронный стек (FastAPI, SQLAlchemy, asyncpg) для обеспечения высокой пропускной способности.
*   **Атомарные счетчики:** Для исключения проблем **Race Condition** при обновлении статистики переходов используется сырой SQL-запрос `UPDATE ... SET visit_counts = visit_counts + 1`, что гарантирует точность данных под нагрузкой.
*   **Надежная генерация ID:** Использование библиотеки `shortuuid` с дополнительной проверкой на коллизии в базе данных.
*   **Контейнеризация:** Проект полностью готов к деплою через Docker и Docker Compose.
*   **Makefile:** Настроены сокращения для всех основных команд разработки.

---

### 1. Подготовка окружения
Склонируйте репозиторий:
```bash
git clone https://github.com/YesCodeUser/url-shortener.git
```
В корне проекта создайте виртуальное окружение. Для windows:
```bash
python -m venv .venv
```
Для Linux/macOS:
```bash
python3 -m venv .venv
```
Активируйте виртуальное окружение. Для windows:
```bash
.venv\Scripts\activate
```
Для Linux/macOS:
```bash
source .venv/bin/activate
```
Установите все зависимости:
```bash
pip install -r requirements.txt
```

Создайте в корне проекта файл .env.docker и заполните его следующим образом:
```bash
POSTGRES_USER=ваш_юзер
POSTGRES_PASSWORD=ваш_пароль
POSTGRES_HOST=postgres или иной хост
POSTGRES_PORT=5432 или иной порт
POSTGRES_DB=ваша_БД
```

---

### 2. Запуск приложения
Все окружение (API + Database) поднимается одной командой:
```bash
make up
```
Приложение будет доступно по адресу: http://localhost:8000

---

### 3. Миграции
При первом запуске необходимо применить миграции к базе данных:
```bash
make migrate
```

---

### 4. Тестирование и Линтинг
Для проекта реализована система автоматизированного тестирования с транзакционной изоляцией. Каждый тест выполняется внутри своей транзакции, которая откатывается по завершении. Это гарантирует чистоту базы данных и независимость тестов.

Запуск тестов:
```bash
make test
```
Запуск автоформатирования из корня:
```bash
black .
```
Запуск линтера из корня:
```bash
ruff check .
```

---

### 5. Документация и API
После запуска сервера интерактивная документация доступна по адресу: http://localhost:8000/docs

Основные эндпоинты:
- POST /shorten — Создание короткой ссылки.
- GET /{short_id} — Редирект на оригинальный URL с увеличением счетчика.
- GET /stats/{short_id} — Получение статистики переходов.
